# Object-Oriented Programming Interview Questions

## 1. What are the four main principles of object-oriented programming? Explain each with an example.

- **Encapsulation:** Hides data and allows controlled access. Example: Bank account with private balance.
- **Inheritance:** A child class inherits properties from a parent class. Example: Dog inherits from Animal.
- **Polymorphism:** One method can perform different actions. Example: `draw()` works differently for Circle and Rectangle.
- **Abstraction:** Hides implementation details and shows only essential features. Example: Driving a car without knowing the engine.

---

## 2. What is the difference between a class and an object?

- **Class:** A blueprint or template.
- **Object:** An instance of a class.

Example: `Student` is a class, while `student1` is an object.

---

## 3. What is encapsulation, and why is it important?

Encapsulation keeps data private and provides methods to access it.

**Importance:** Protects data and improves security.

---

## 4. What is inheritance? What are its advantages and disadvantages?

Inheritance allows one class to use the properties and methods of another class.

**Advantages:**
- Code reuse
- Easy maintenance

**Disadvantages:**
- Tight coupling
- Can make code complex

---

## 5. What is polymorphism? Explain compile-time and runtime polymorphism.

Polymorphism allows the same method to behave differently.

- **Compile-time:** Method overloading.
- **Runtime:** Method overriding.

---

## 6. What is abstraction, and how is it different from encapsulation?

- **Abstraction:** Hides implementation details.
- **Encapsulation:** Hides data and controls access.

---

## 7. What is the difference between an abstract class and an interface?

- **Abstract class:** Can have abstract and normal methods.
- **Interface:** Defines methods that classes must implement.

---

## 8. What is method overloading, and how is it different from method overriding?

- **Method Overloading:** Same method name with different parameters.
- **Method Overriding:** Child class provides a new implementation of a parent class method.

---

## 9. What is the difference between composition and inheritance? When should composition be preferred?

- **Inheritance:** Represents an "is-a" relationship.
- **Composition:** Represents a "has-a" relationship.

Composition is preferred because it provides more flexibility and loose coupling.

---

## 10. What are access modifiers, and how do they help control access to data and behavior?

Access modifiers control the visibility of classes, methods, and variables.

**Types:**
- Public
- Protected
- Default
- Private

They help protect data and improve security.
