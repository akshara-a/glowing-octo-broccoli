✏️ No positioned parent? If the parent has no positioning context (relative/absolute/fixed/sticky), the absolutely-positioned child escapes and positions itself relative to the nearest positioned ancestor — usually <body>. Always set position: relative on the parent to contain the child.

Part D – Fixed Position

The blue ▲ "Back to Top" button and green 💬 chat button are position: fixed at the bottom-right. The dark header at the top is also fixed. Scroll to verify.

✏️ position: fixed removes the element from normal flow and pins it to the viewport. It never moves no matter how much you scroll. Use for persistent UI: headers, FABs, cookie banners.

Part E – Sticky Position

The dark navigation bar above uses position: sticky; top: 56px. It scrolls with the page until it hits that threshold, then sticks.

✏️ Fixed vs Sticky:
• fixed → always relative to the viewport; completely out of flow; stays even after parent ends.
• sticky → stays in flow; scrolls normally until threshold is reached; stops sticking once its parent scrolls past.
Sticky is ideal for navbars and table headers that should only persist in context.

Part F – Z-Index

✏️ Z-index stacking: Higher z-index = rendered above lower values. Only works on positioned elements (anything except static). When two elements share the same z-index, the one later in the HTML source is on top.

Challenge Questions – Answers


#QuestionAnswer
1 Default value of position? static
2 Removes element from document flow? absolute and fixed
3 Stays visible during scrolling? fixed (and sticky within its parent)
4 Requires top/left/right/bottom to stick? sticky — without e.g. top: 0 it won't stick
5 Same z-index on multiple elements? The element later in the HTML source is rendered on top
6 Absolute vs Relative — when? Use absolute to break out of flow and pin precisely (badges, tooltips). Use relative for small visual nudges while keeping the layout slot.
7a Real-world: Static Regular article paragraphs and headings in a blog post.
7b Real-world: Relative A button icon shifted 2px up for optical alignment without affecting nearby text.
7c Real-world: Absolute A notification dot pinned to a profile avatar's top-right corner.
7d Real-world: Fixed A cookie consent banner always visible at the bottom of the screen.
7e Real-world: Sticky A table's header row that sticks to the top while scrolling through hundreds of rows.
