1. Encapsulation , inheritance , polymorphism , abstraction

2. A class is a blueprint of object
   An object is an actual instance of class

3.    Encapsulation means wrapping data and methods into a single unit class.

It is important because it improves security and protects data from unauthorized access.

4. A class which acquires all the properties and behaviours of it's parent class
advantages - code reusability

5. polymorphism means doing the same method in different ways

6. Abstraction means hiding implementation details and showing only necessary features.

7. Abstract Class:
Can have both abstract and normal methods.
Supports partial implementation.

Interface:
Contains only method declarations (in basic OOP concepts).
Used for complete abstraction.

8. method overloading : the same method which have diffrent parameters
method overriding : Child class provides a new implementation of a parent class method.
